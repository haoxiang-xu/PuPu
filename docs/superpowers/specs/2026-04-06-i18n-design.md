# PuPu UI Internationalization (i18n) Design

**Date:** 2026-04-06
**Status:** Approved

## Goal

Add multi-language support to PuPu's UI, starting with English and Simplified Chinese. The UI language is independent of the LLM language.

## Approach

Lightweight, self-built solution: JSON translation files + a custom `useTranslation()` hook. No external i18n libraries.

## Translation Files

Location: `src/locales/`

```
src/locales/
  en.json        — English (default / fallback)
  zh-CN.json     — Simplified Chinese
```

Structure: nested by component/feature area.

```json
{
  "appearance": {
    "title": "Appearance",
    "theme_mode": "Theme Mode",
    "theme_mode_desc": "Choose between light and dark mode",
    "language": "Language",
    "language_desc": "Choose display language"
  },
  "settings": {
    "model_providers": "Model Providers",
    "workspaces": "Workspaces"
  }
}
```

Access via dot notation: `t("appearance.title")`.

## `useTranslation` Hook

Location: `src/BUILTIN_COMPONENTs/mini_react/use_translation.js`

```js
const { t, locale, setLocale } = useTranslation();
```

- `t(key)` — resolve key against current locale JSON; fallback to English; fallback to key itself
- `locale` — current locale string (`"en"` | `"zh-CN"`)
- `setLocale(val)` — update locale in ConfigContext

## ConfigContext Integration

### Storage

Language preference stored in `localStorage("settings")` under `appearance.locale`:

```json
{
  "appearance": {
    "theme_mode": "dark_mode",
    "locale": "zh-CN"
  }
}
```

Uses existing `saveSettingsStorage("appearance", { locale })` pattern.

### Runtime State

In `container.js`:
- Read initial locale from `settings.appearance.locale`, default `"en"`
- Add `locale` + `setLocale` state
- Include in `ThemeContext` value (alongside theme-related appearance state)
- Persist on change via `saveSettingsStorage`

## Appearance Settings UI

In `appearance.js`, add a new `SettingsRow` below Theme Mode:

```
Label:       "Language" / "语言"
Description: "Choose display language" / "选择显示语言"
Control:     <Select> with options:
             - { value: "en",    label: "English" }
             - { value: "zh-CN", label: "中文" }
```

Reuses existing `SettingsRow` + `Select` components.

## Translation Scope

### Translate:
- Settings pages (appearance, model providers, memory, local storage, token usage, update, dev)
- Side menu labels
- Chat UI (placeholder text, button labels, status messages)
- Init setup flow
- Common UI (confirm/cancel buttons, error messages, tooltips)

### Do NOT translate:
- LLM system prompts (independent of UI language)
- Provider names ("OpenAI", "Anthropic", etc.)
- Code and technical identifiers

## Migration Strategy

Replace hardcoded strings with `t()` calls incrementally, component by component.
