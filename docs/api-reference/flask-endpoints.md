# Flask Endpoints

> Complete HTTP API reference for the Python Flask sidecar (`unchain_runtime/server`).

---

## Overview

The Flask sidecar runs on `127.0.0.1`, port range `5879-5895`, with a per-session auth token (256-bit hex) passed via `x-unchain-auth` header.

All endpoints require authorization via `@before_request` middleware except `/health`.

## Chat

| Method | Path | Description |
|--------|------|-------------|
| POST | `/chat/stream` | Chat streaming V1 (legacy, simple events) |
| POST | `/chat/stream/v2` | Chat streaming V2 (frame-based SSE protocol) |
| POST | `/chat/stream/v3` | Chat streaming V3 (typed RuntimeEvent SSE protocol) |
| POST | `/chat/tool/confirmation` | Submit tool confirmation response |

### `/chat/stream/v2` and `/chat/stream/v3` Request Body

```json
{
  "message": "Hello",
  "history": [
    { "role": "user", "content": "Previous question" },
    { "role": "assistant", "content": "Previous answer" }
  ],
  "threadId": "thread-123",
  "attachments": [],
  "selectedToolkits": ["workspace_toolkit"],
  "workspace_roots": ["/path/to/workspace"],
  "options": {
    "model": "gpt-4.1",
    "provider": "openai",
    "api_key": "...",
    "memory_enabled": true,
    "memory_session_id": "session-123",
    "memory_embedding_provider": "openai",
    "system_prompt_v2_sections": {
      "personality": "...",
      "rules": "..."
    }
  }
}
```

### `/chat/stream/v2` SSE Response

See [Request Flow & Streaming](../architecture/request-flow-and-streaming.md) for the complete SSE frame protocol.

### `/chat/stream/v3` SSE Response

V3 uses the same SSE transport as V2 but emits `event: runtime_event` payloads. The stream terminates with `event: done` and may include diagnostics or an error summary.

See [Runtime Events V3](../architecture/runtime-events-v3.md) for the event schema and frontend state path.

### `/chat/tool/confirmation` Request Body

```json
{
  "confirmation_id": "...",
  "approved": true
}
```

## Authentication

All requests (except `/health`) must include:

```
x-unchain-auth: <256-bit-hex-token>
```

The token is generated by the Electron main process at startup and passed to both the Flask sidecar and the IPC layer. A `miso_auth` query parameter is still accepted by a few asset/debug routes for legacy compatibility.

---

## Error Response Format

```json
{
  "error": {
    "code": "unauthorized",
    "message": "Invalid auth token"
  }
}
```

Common error codes: `unauthorized`, `not_found`, `bad_request`, `internal_error`.

## Route Modules

The routes are organized across multiple files:

| Module | Routes |
|--------|--------|
| `routes.py` | Legacy monolith (being split) |
| `route_blueprint.py` | Blueprint registration |
| `route_chat.py` | `/chat/*` |
| `route_memory.py` | `/memory/*` |
| `route_characters.py` | `/characters/*` |
| `route_catalog.py` | `/models/*`, `/toolkits/*` |
| `route_projection.py` | Projection endpoints |
| `route_auth.py` | Auth middleware |
