关键决策(锁定)

  • 按 PR 切片而不是一次性合并 — P4 是唯一的"危险步骤",其它 6 步都是纯新增或只读
  • P3 提交时 registry 已经可用但没人消费 — 方便单独验证 boot 顺序与 store 加载,不会影响 chat
  • P4 不改默认行为 — 没传 primary_agent_id 时回退到 pupu_developer,现有 session 零感知
  • 前端 P6/P7 分开 — P6 只做数据通路(配了就能在 devtools 里看到 agents 列表),P7 才接 UI
  • E2E smoke 放在 P7 之后补 — 之前的 PR 用单测 + 现有 E2E 兜底
