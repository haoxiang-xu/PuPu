核心决策摘要(锁定)

  • Scope — 统一池;所有 agent 天然平等,不区分 primary/sub 角色
  • 来源 — 混合:内置走 prompts/agents/*.py,用户自建走 JSON store
  • Character 关系 — 完全独立,character_store 不动,新增 agent_store 平行存在
  • 运行时选择 — 用户可在 chat 里选 primary,缺省走 pupu_developer
  • Prompt — 6 段(identity / capability / workflow / delegation / constraints / fallback)
  • Toolkit — 硬约束(agent 的 allowed_toolkits 覆盖会话级选择)
  • Delegate — 默认 ["*"](互通),由 max_depth=2 防失控

关键不变量

  • 内置 agent 的权威源依然是 prompts/agents/*.py,JSON store 只存用户自建
  • Registry 在 Flask 启动时初始化一次,内置 agent 永久注册,用户 agent 从 store 懒加载
  • Registry 是后端的单一事实源,前端通过 /agents API 获取快照,不做本地缓存式合并
  • 内置 agent 在 registry 里标 is_builtin=true,前端 UI 只读
  • 运行时没有"硬编码 developer 路径" —— 所有 agent 走 _build_agent_from_spec
